## 2026-04-29 — Phase G4 — Drawio editor (modifica originale + signed URL)

**Trigger**: G2 ha posato il blob cifrato, G3.b crea blob nuovi. Manca il
flow "apri mappa esistente per modifica" — necessario per chiudere il
ciclo CRUD drawio. G4 introduce signed URL HMAC + `MapsController::update`
con optimistic concurrency + frontend hook su `.fm-item-edit`.

### Aggiunte
- **MapSignedUrlService**: mint/verify token HMAC-SHA256 su payload
  `{i:content_id, m:view|copy, e:exp}`, base64url encoded. Riusa
  `STORAGE_SIGNING_SECRET` (no nuova chiave). TTL clamp [60, 3600],
  default 600.
- **Endpoint MapsController nuovi**:
  - `GET /api/maps/{id}/signed-url?mode=view|copy` (auth+teacher) →
    permission check via `MapPermissionService::canView/canCopy` →
    mint signed URL TTL 600s.
  - `GET /api/maps/dl?t=&s=` (pubblico, signed URL = auth) → verifica
    HMAC + TTL → load row → MapBlobStore::get(owner_teacher_id) →
    stream decrypted con `Cache-Control: private, no-store`.
  - `POST /api/maps/{id}/update` (auth+teacher+csrf+rate, owner only) →
    sanity check XML mxfile root → optimistic concurrency su
    `map_version` → re-encrypt blob (stesso ulid) + bump version.
    Mismatch versione → 409 con `server_version` in body.
- **Frontend `drawio-editor.js`**:
  - `openDrawioEditor({contentId, mode})` — apre embed.diagrams.net
    full-screen, fetcha XML decifrato via signed URL, postMessage bus
    init→load + save→post update. Mode 'view' nasconde save button.
- **Hook `sidepage-inline-actions.js`**:
  - `wireItemActions` intercept del click `.fm-item-edit` per type=mappa
    con `map_blob_path` → apre drawio editor invece del modal standard.
    Mappe legacy (link only) continuano via modal classico.

### Sicurezza
- HMAC-SHA256 signed URL: tampering = `signature_mismatch` (rejected).
- TTL hard clamp 60-3600s.
- Permission check al MINT (server side), non alla CONSUMPTION:
  l'URL e' "pre-autorizzato" come signed URL S3 presigned.
- Optimistic concurrency: 2 tab editing → seconda save → 409 (no lost
  update).
- Owner-only update (canEdit). Cross-teacher save rejected.

### Test
- Smoke: mint → verify → round-trip OK; tampering rejected
  (`signature_mismatch`).
- Build vite green (`bootstrap.{hash}.js`).
- Manual end-to-end test: in fase post-merge G4 PR.

### Note
- Mode 'copy' su drawio editor: G4 implementa il flow base (POST
  `/api/maps` con `drawio_native`), ma `parent_map_id` metadata
  population resta TODO (richiede ALTER metadata.mappa.parent_map_id
  in MapsController::create — G4.b o G7).
- Routes flat (no group): `/api/maps/{id}/signed-url` su middleware
  auth+teacher (no csrf, GET); `/api/maps/dl` PUBLIC (signed URL =
  auth); `/api/maps/{id}/update` group csrf+rate (POST mutation).

## 2026-04-29 — Phase G2 — Mappe: storage locale cifrato + sharing granulare

**Trigger**: G1.a ha posato l'OAuth ma le mappe non sono ancora persistite
lato server. G2 introduce il blob locale cifrato (single source of truth)
e il modello di sharing fine-grained richiesto dal piano (cross-teacher
opt-in con permission view/copy).

### Aggiunte
- Migration `020_map_blob_and_shares.sql`:
  - ALTER `teacher_content` ADD `map_blob_path`, `map_mime`, `map_size`,
    `map_drive_id`, `map_origin` (enum drive_legacy/upload/drawio_native/
    drive_orphan), `map_is_public`, `map_version` (optimistic concurrency).
  - Nuova tabella `map_shares` (content_id, scope_type ∈ {institute,
    class, student, teacher}, scope_value, permission ∈ {view, copy}).
- Service `App\Services\Maps\MapBlobStore`: put/get/delete/exists su
  `storage/maps_enc/{teacher_id}/{ulid}.bin` con layout binario
  `[kv 2B BE][iv 12B][gcm tag 16B][ciphertext]`. Atomic write via temp +
  rename. Path traversal guard preg `^\d+/[0-9A-Z]{26}\.bin$`.
- Repository `App\Repositories\MapShareRepository`: grant/revoke/
  revokeAll/listForContent/bestPermissionFor con UPSERT
  (content_id, scope_type, scope_value).
- Service `App\Services\Maps\MapPermissionService`: canView/canCopy/
  canEdit/describeAccess con priority: owner > public > shares > class.
  Class scope richiede context (institute_id+indirizzo+classe della
  sessione, NON derivato da `users` perche' studenti non hanno
  attributo classe persistente).

### Test
- 12 unit test in `tests/Unit/Maps/` (path guard + scope/permission
  validation) — 100% pass.
- Live smoke: round-trip 53B plaintext → 83B ciphertext → decrypt OK
  per teacher_id=77.

### Note
- Crypto-shredding O(1) si estende automaticamente ai blob: DELETE
  `teacher_keys` row → tutti i `map_blob_path` del docente diventano

### TeacherCryptoService

- Encrypt body con KEK → AES-256-GCM, iv random per row, tag 16B
- Logging append-only: ogni op (encrypt/decrypt/shred/rotate/wrap/unwrap) → crypto_access_log

### B2 — Race fix `instance_key` (atomic ON DUPLICATE KEY UPDATE)

**Bug pre-esistente**: `instancesCreate` aveva una race window:
1. `listInstances($tid, $id)` → array `taken` keys
2. PHP loop `while (in_array($key, $taken))` $key = base.'-'.$i++
3. `INSERT IGNORE` finale

Due request concorrenti con stesso label entrambe passavano lo step 2 con la stessa key (no collision visible yet). INSERT race → una vinceva, l'altra otteneva IGNORE silenzioso. Entrambe API "ok" con la stessa instance_key → caller all'oscuro del conflitto.

**Fix**:

`app/Services/Risdoc/OverrideRepository.php`:
- Cambiato `INSERT IGNORE` → `INSERT ... ON DUPLICATE KEY UPDATE id = id`
- Tipo ritorno `void` → `bool` (true se inserito, false se già esiste)
- `$stmt->rowCount() === 1` discriminante: 1 = INSERT eseguito, 0 = ON DUPLICATE no-op

`app/Controllers/Risdoc/TemplateController.php`:
- Sostituito PHP `while`-loop pre-INSERT con retry-loop atomico fino a 50 iterazioni
- Ogni iterazione tenta INSERT atomico; se collision (rowCount=0) genera key successiva con suffisso
- 409 `instance_key_collision_exhausted` se 50 iterazioni esauriscono (paranoid; in pratica 1-2 collisioni max)

## 2026-05-01 — Phase G19.1 — TEX raw LaTeX preservato (no Unicode glyphs)

**Trigger**: utente downloads `verifica_113.zip`, apre `.tex`, vede:
```
\item (1.0 p) ⎧⎪⎨⎪⎩1𝑥+1−2𝑦=7𝑥+12𝑥𝑦+2𝑦𝑦+63−𝑥=2
```
Invece di:
```
\item (1.0 p) \(\begin{cases}\dfrac{1}{x+1}-\dfrac{2}{y}=\dfrac{7x+1}{2xy+2y}\\\dfrac{y+6}{3}-x=2\end{cases}\)
```

### Cause

1. **Frontend** (`buildSelectionFromDOM`) inviava
   `el.querySelector(".collex").innerHTML` = HTML con MathJax-rendered
   `<mjx-container>` containing Unicode glyphs ⎧⎪⎨⎪⎩.
2. **Backend** (`Sanitizer::stripHtml`) chiamava `strip_tags` che lasciava
   solo `textContent` (Unicode glyphs).
3. **Backend** (`Sanitizer::escapePlain`) avrebbe fatto double-escape se
   il client avesse mandato LaTeX raw (`\` → `\textbackslash{}`).

### Fix

- **Client**: `extractItemHtml(el)` raccoglie `data-raw` da
  `.fm-text/.fm-latex` (preservati da ContractRenderer) in document-order.
  Concatena conservando block (`<p><div>`) vs inline (`<span>`).
- **Backend**: nuovo `Sanitizer::latexPassthrough` (strip tags HTML +
  collassa blank lines, NO escape). Sostituisce `escapePlain(stripHtml(...))`
  in `TexBuilder.php` (problem text) e `TableRenderer.php` (item.html
  per Collect/RMulti/VF).

### Test E2E

`g18_real_user_genera.spec.js` aggiunge guard contro regressioni:
- `expect(tex).not.toMatch(/[⎧⎪⎨⎩⎫⎬⎭⎰⎱]/)` — nessun glyph Unicode
- `expect(tex).not.toContain("\\textbackslash{}")` — niente double-escape
