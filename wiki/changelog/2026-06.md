## 2026-06-05 — Hardening client: WAF JSON-per-XHR, CSRF centralizzato, de-jQuery, CSS-in-JS zero

Sessione di hardening + centralizzazione lato client/sicurezza. Vedi
[security-notes](../security-notes.md), [waf](../waf.md),
[docs/security/track6-7-csp-csrf-hardening.md](../../docs/security/track6-7-csp-csrf-hardening.md).

### WAF — risposte JSON per richieste XHR/API
- `WafMiddleware`: flag centralizzato `$expectsJson` (`wantsJson()` o path
  `/api/*`). Challenge e block ritornano **JSON 403** alle XHR
  (`{code:'waf_challenge',reload:true}` / `{error:'request_blocked'}`) invece
  della pagina HTML — **decisione di sicurezza invariata** (sempre 403,
  fail-closed). Fixa il crash `Uncaught SyntaxError: Unexpected token '<'` sulle
  `fetch`.
- Client: `dom-utils.assertJson` riconosce `waf_challenge` (JSON + HTML
  difensiva) e fa auto-reload one-shot (debounce 30s) per rinnovare il cookie
  `waf_session`.
- Test: `tests/Unit/WafMiddlewareJsonResponseTest.php`.

### CSRF — centralizzazione totale
- Unica fonte client `dom-utils.fetchCsrf` (cache 60s). Rimossi/aliasati ~tutti
  gli helper locali (`getCsrf`/`csrf`/`bsCsrf`/`_getCsrf`/`csrfToken`) in view
  inline, `tikz-*`, `checkin-handlers`, adapter pt-document, sidepage,
  `verifica-*`, `admin-*`. Unico `fetch('/auth/csrf')` rimasto = canonico in
  `dom-utils.js`.

### De-jQuery / de-shim
- Eliminato `js/modules/core/ajax-compat.js` (`$.ajax`-like `done/fail/always`):
  8 moduli migrati a `fetch`/`fetchJson` vanilla (`then/catch/finally`):
  `dom-manager`, `content-processor`, `editor-system`, `utilities`,
  `event-handler`, `google-apps-script`, `ui-comp`, `table-manager`.
- Confermato zero jQuery nel runtime (libreria non caricata; `$` = alias
  `querySelector`).

### CSS-in-JS azzerato (ADR-023 Fase 5)
- Ultima iniezione runtime (`bootstrap-compat.injectSyncStyles`) → modulo
  `css/modules/_sync-status.css` (import in `components.css`). CI `no-css-in-js`
  verde.

### Profilo docente + componente autocomplete
- Fix area `/area-docente/profilo`: tab Classi/Materie (bug `.fm-d-none`),
  selettore classe scopato all'istituto attivo, feedback "Già collegato"
  onesto, nome MIUR ufficiale al posto dell'`header_label`.
- Nuovo componente vanilla riusabile e accessibile (WCAG 2.2 AA)
  `js/modules/ui/autocomplete.js` (combobox a tema dark) — sostituisce il
  `<datalist>` nativo.

### Audit `/admin` (esito positivo, nessuna modifica)
- Routing dietro `auth + role:admin`; mutazioni in gruppi `csrf+rate`;
  operazioni sensibili `super_admin_required` (+ `audit_reason`); gruppi senza
  CSRF contengono solo GET. SQLi coperto (prepared + whitelist
  `/admin/logs/api/{table}`); XSS coperto (`esc`/`escAttr`; nessun `<?= $var ?>`
  grezzo). `/api/admin/*` in `shouldBypass` del WAF.

### Gate
- `no-css-in-js`, `csp:no-inline-handlers`, `eslint --quiet` (0 errori),
  `vite build` — tutti verdi.
